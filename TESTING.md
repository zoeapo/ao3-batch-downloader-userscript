# 人工测试清单

当前测试环境目标：Chrome + Tampermonkey。

先确认 Tampermonkey 下载模式为 Browser API，并允许 EPUB、PDF、TXT、JSON 扩展名。

## 1. 安装和界面

- [ ] 安装脚本后，AO3 页面右下角出现“AO3 批量下载”。
- [ ] 点击按钮后中文面板正常打开。
- [ ] 关闭按钮正常。
- [ ] Tampermonkey 菜单中的“打开 AO3 批量下载面板”正常。
- [ ] 刷新页面后设置仍然保留。

## 2. 单篇作品

选择一篇公开、至少两章的作品：

- [ ] 在作品任意章节页面点击“扫描全部分页”。
- [ ] 发现数量为 1。
- [ ] 导出的 TXT 链接是 `https://当前AO3域名/works/数字`。
- [ ] 下载 EPUB 成功。
- [ ] 下载 PDF 成功。
- [ ] 文件包含当前全部章节。
- [ ] 文件名与 AO3 手动下载时使用的官方文件名一致。

## 3. 标签或搜索列表分页

选择至少有两页结果的标签页或搜索页：

- [ ] 从第二页或更后面的页面开始扫描，脚本仍从第一页扫描。
- [ ] 页数持续增加，直到最后一页。
- [ ] 相同作品不会重复计数。
- [ ] “导出全部链接 TXT”每行一个标准作品链接。

## 4. 作者、书签和系列

分别测试：

- [ ] 作者作品列表。
- [ ] 用户书签列表。
- [ ] 系列页面。
- [ ] 只有简单作品链接的订阅页面。

确认作品数量与页面内容大致一致，并检查日志中是否出现无法识别的页面。

## 5. 筛选

准备同时包含完结和未完结、不同字数、不同分级的列表：

- [ ] 仅完结。
- [ ] 仅未完结。
- [ ] 最低字数。
- [ ] 最高字数。
- [ ] 分级。
- [ ] 分类。
- [ ] Fandom 完整匹配。
- [ ] Relationship 完整匹配。
- [ ] Additional Tag 包含文字。
- [ ] 排除 Warning。
- [ ] 排除 Relationship。
- [ ] “命中任意一项”。
- [ ] “必须全部命中”。

每次修改筛选后点击“重新应用筛选”。如果新增语言筛选或当前页面只有简单链接，重新扫描全部分页。

## 6. 语言筛选

- [ ] 填写当前作品实际语言。
- [ ] 扫描日志提示正在补充语言信息。
- [ ] 扫描请求之间遵守设置间隔。
- [ ] 不匹配语言的作品被排除。

## 7. 下载记录

选择一部作品，同时下载 EPUB 和 PDF：

- [ ] 第一次两种格式均下载成功。
- [ ] 开启“跳过已有成功记录”后再次运行，两种格式都跳过。
- [ ] 清除其中一种格式的记录或制造一种格式失败，再次运行时只重试缺失格式。
- [ ] 导出 JSON 能看到作品编号、格式、状态、时间和文件名。
- [ ] 清空记录后可以重新下载。

## 8. 暂停和停止

选择多部作品：

- [ ] 点击暂停后，新文件不再开始下载。
- [ ] 正在进行的文件允许完成。
- [ ] 点击继续后任务恢复。
- [ ] 点击停止后活动下载被取消，队列停止。

## 9. 请求限制与错误

不建议主动制造 AO3 访问限制。遇到实际错误时检查：

- [ ] HTTP 429 时日志显示暂停秒数。
- [ ] HTTP 5xx 时有限次数重试。
- [ ] 已删除作品记为失败，任务继续。
- [ ] 未登录的锁定作品记为失败，任务继续。
- [ ] 不可访问隐藏合集记为失败，任务继续。
- [ ] 某一种格式缺少链接时只影响该格式。

## 10. Chrome 权限

- [ ] 首次多文件下载时允许批量下载。
- [ ] 禁止 Tampermonkey 下载权限时，日志能显示 `not_enabled` 或 `not_permitted`。
- [ ] EPUB/PDF 不在扩展名白名单时，日志能显示 `not_whitelisted`。
